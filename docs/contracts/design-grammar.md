# Design Grammar Contract

## Non-Negotiable Structure
Every project route must present information in this sequence:
1. `Hero`
2. `BLUF`
3. `Data Integrity`
4. `Interactive Chapter A` (SSR shell)
5. `Interactive Chapter B` (lazy gate + client interactive)
6. `Decision Console`
7. `Assumptions`

## Per-Project Differentiation
Each project must have route-specific differences in:
1. Hero composition and typography hierarchy.
2. Primary chart family and annotation style.
3. Accent token application (`market-competition`, `forensic-risk`, etc.).
4. Motion pacing and reveal cadence.

## Reuse Constraints
Allowed shared primitives:
- `ProjectFrame`
- `StoryChapterShell`
- `DecisionConsole`
- `AssumptionsDrawer`

Not allowed:
- identical module order with only color swaps.
- one shared chart grammar used as the primary chart across all routes.

## Annotation Density
Per project, minimum narrative density:
1. `2+` interactive analytical charts.
2. `1+` annotation rail with evidence references.
3. `1+` decision evidence panel with confidence context.
